## 文档用途

本文档用于集中维护当前课题的工程状态、数据版本、实验结论、关键脚本、当前瓶颈和下一阶段任务。它不是论文正文，也不是汇报讲稿，而是项目连续推进时的“总账”。后续若切换窗口、重启环境或更换服务器，应先读本文件，再根据文末日志索引追溯具体细节。

当前总览更新时间为 `2026-04-09`。

## 一、课题目标与当前边界

当前课题的核心目标是：基于稀疏传感器测点的浓度-时间序列，反演受限空间中的三维氢气浓度场时空演化，并进一步估计泄漏源位置与泄漏率大小。

当前边界已经明确：

- 主任务是三维浓度场反演。
- 源位置定位与泄漏率估计是并行推进的扩展输出。
- 当前不包含危险区判断或风险区划。
- 当前数据仍主要来自 CFD，多工况真实实验数据尚未正式并入训练闭环。

## 四、正式主结果

### 1. 三维浓度场反演基线

基于 38 组数据，已经完成正式 holdout 训练与评估：

- 训练集 `31` 组
- 测试集 `7` 组
- holdout 位置为 `(400,0,0)`

当前正式主结果可概括为：

- `global_rmse = 0.002540`
- `global_mae = 0.000700`
- `global_rel_l1_active_mean = 4.434`
- `global_rel_l2 = 8.417`
- `mass_mean_rel_error = 4.430`

低泄漏率子集 `50/100 mL·min⁻¹` 明显更难，其误差显著高于整体平均，因此后续评估必须同时报告“整体指标”和“低流量子集指标”。

### 2. 源参数估计基线

源参数估计已经从早期的峰值搜索推进到正式监督回归。当前三种模式比较结果是：

- `sensor`：当前最优正式基线
- `core`：明显弱于 `sensor`
- `hybrid`：当前样本规模下未优于 `sensor`

当前 `sensor` 基线结果为：

- 源位置平均误差 `160.37 mm`
- 泄漏率平均相对误差 `1.71%`
- 泄漏率 MAE `3.16 mL/min`

因此短期内已经冻结：

- 不再用峰值搜索作为主结果
- 正式源参数方案以 `sensor` 回归器为准

## 五、当前最重要的技术判断

### 1. 当前最优绝对精度配置

在 `MPDPS` 第二轮调参后，当前绝对精度最优配置已经冻结为：

- `mpdps_weight = 16`
- `total_steps = 50`
- `num_posterior_samples = 2`
- `zeta = 0.03`

对应文件：

- `SDIFT模型/exps/mpdps_best_absolute_holdout400_stage2.json`

注意：这只能称为“当前最优绝对精度配置”，不能解释为“观测依赖最强配置”。

### 2. 观测约束当前仍然太弱

完整 sanity check 已经完成，测试了四种输入：

- `correct`
- `shuffled`
- `zeros`
- `wrong_positions`

它们在当前最优绝对精度配置下的结果几乎重合。这说明：

- 当前瓶颈已经不是同类 `MPDPS` 超参数没扫够
- 而是 `message_passing_DPS.py` 中的观测注入机制本身过弱

因此，下一阶段不应继续优先扫：

- `mpdps_weight`
- `zeta`
- `total_steps`
- `posterior_samples`

而应转向：

- 记录 `llk_grad_norm / diffusion_update_norm / obs_update_norm`
- 分析观测项是否被时间核传播与基函数投影过度平滑
- 尝试观测项归一化、自适应放大或更直接的条件注入

### 3. 导师要求的两条曲线暂不应直接重跑

在观测项没有真正起作用前，直接重跑下面两条曲线会被机制问题污染：

- 传感器数量 / 观测时长影响
- 训练数据量 `6/12/24/31` 的多次重复分层实验

因此它们不是取消，而是延后到观测注入机制修复之后再做。

### 4. 尺寸泛化目前仍无数据支撑

虽然数据总量已经从 38 组增加到 49 组，但新增数据仍属于同一箱体尺寸。  
所以导师要求的“尺寸稍大或稍小场景迁移”目前仍没有正式实验支撑。

后续补数优先级应为：

1. 补 `(200,0,0)` 那 7 组未完成工况
2. 补缺失位置的 `50/100 mL·min⁻¹` 低流量样本
3. 优先新增不同箱体尺寸的 CFD 数据

## 九、观测注入机制改造状态

基于前述机制诊断，当前已经不再把问题归因于 `mpdps_weight / zeta / total_steps / posterior_samples` 等超参数未扫够，而是正式归因于 `message_passing_DPS.py` 中观测注入机制过弱。为此，本地已完成一轮代码级改造，并保留了与旧实验的兼容性。

当前 `message_passing_DPS.py` 已新增三种观测注入工作模式：
- `legacy`：保留旧版 `obs_update = zeta/(i+1) * llk_grad`
- `adaptive_ratio`：根据当前步 `diffusion_update_norm` 与 `llk_grad_norm` 自适应计算观测缩放，使观测修正达到预设比例
- `direct_inner`：在单个扩散步内部执行多次观测修正，并在内层循环中重新调用去噪网络估计 core，形成更直接的条件注入

同时，诊断信息也已扩展。每一步现在会记录 `obs_base_scale`、`obs_target_scale`、`obs_inner_scales`、`obs_inner_grad_norms`、`obs_inner_update_norms`、`obs_inner_total_update_norm` 等信息，便于后续精确判断“观测项是否真的被放大到与扩散项可比的量级”。

相关外围脚本 `run_holdout_reconstruction_eval.py`、`run_observation_sanity_check.py`、`run_mpdps_observation_tuning.py` 和 `run_mpdps_diagnostic_case.py` 也已同步支持这些新参数，后续服务器实验可以直接切换到新模式，无需再次手工改脚本。

下一轮服务器实验的候选配置文件已准备为：
- `SDIFT模型/exps/mpdps_direct_inner_candidate_holdout400.json`

当前最合理的后续顺序是：
1. 先在服务器上用 `direct_inner` 候选配置重跑单 case 诊断，确认 `correct` 是否开始明显优于 `shuffled / zeros / wrong_positions`
2. 若观测项首次真正拉开，再重跑导师要求的“传感器数量 / 观测时长”和“训练数据量多次重复”两条正式曲线
3. 若仍然拉不开，再继续改观测项归一化和条件注入公式，而不是回退到旧版超参数扫描
4. 在机制修复完成前，49 组数据暂不直接启动新一轮正式大训练

当前该阶段已经进一步推进。`direct_inner` 的服务器单 case 诊断已经完成，并得到更积极的结论：观测更新与扩散更新的范数比值已从旧机制下的约 `0.001x` 提升到均值约 `0.42`、最大约 `0.55`，说明观测项已经进入与扩散项同一数量级。高泄漏率代表工况 `case_0027 (1000 mL/min)` 上，`correct` 已经明显优于 `shuffled / zeros / wrong_positions`；低泄漏率代表工况 `case_0033 (50 mL/min)` 上，`correct` 已明显优于 `zeros` 和 `wrong_positions`，但与 `shuffled` 的差距仍较小。这意味着观测注入机制已经从“基本无效”推进到“高流量有效、低流量仍偏弱”的状态。

据此，当前后续顺序应调整为：
1. 不再回到旧版 `MPDPS` 超参数盲扫。
2. 先在完整 `holdout_400_0_0` 的 7 个测试 case 上，用 `direct_inner` 配置做一次正式 sanity check，确认改进能否在整个未见位置测试集上稳定成立。
3. 如果完整 holdout 上 `correct` 平均优于 `shuffled / zeros / wrong_positions`，则正式恢复导师要求的两类实验：传感器数量 / 观测时长影响，以及训练数据量多次重复分层实验。
4. 在这些正式曲线中，必须同步报告整体指标和 `50/100 mL/min` 低流量子集指标，因为当前证据已经表明低流量仍是最薄弱环节。
5. 主模型层面暂不需要重新训练 `FTM/GPSD`；当前优先事项是先验证新观测机制，再决定是否基于 `49` 组数据启动新一轮正式大训练。
