# 导师汇报摘要 2026-04-13

这份材料不是完整技术报告，而是面向导师的短摘要。目标只有一个：让老师在最短时间内看懂“我们现在到底做出了什么结果、问题卡在哪里、下一步希望他判断什么”。

## 先讲结论

一句话结论可以直接这样说：

本阶段已经不再只是“画图”或“展示方法思路”，而是已经在可控的 CFD 真值下完成了一个可量化的闭环验证：先从未参与训练的 CFD 工况中抽取少量传感器浓度时间序列，再让模型反演三维浓度场，最后回到 CFD 全场真值逐点对比，并给出误差指标。

如果只允许再补一句，那么建议说：

当前最可靠的主结果是三维浓度场反演已经被证明“可验证、可量化”；泄漏率估计已经有初步可用 baseline，泄漏源定位也能做，但精度还不够好，暂时不应讲得和浓度场主线一样成熟。

## 当前状态

当前工作已经从“方法搭起来了”推进到“正式做过闭环验证并能给出数字”的阶段。

第一，数据层面已经完成了一次必要的清洗。原始候选数据一共有 56 组 CFD 工况，但其中 8 组原始浓度列本身就是异常的常数 1，不能代表真实扩散过程。如果把这些数据继续混在训练和评估里，会把结果整体污染。因此当前正式结论只基于清洗后的 48 组有效工况，也就是现在的 `clean48` 数据集。

第二，已经完成了导师最关心的那条验证链：  
CFD 全场真值 → 抽样成传感器输入 → 模型反演三维浓度场 → 回到 CFD 全场逐点对比。

第三，已经做了两类正式测试：

1. 未见泄漏位置测试。也就是拿训练中没见过的位置 `(400,0,0)` 来做闭环验证，测试集 7 组工况，当前 `RMSE = 6.521e-4`，`MAE = 3.717e-4`。这说明模型对“新位置”已经有初步泛化能力。
2. 未见泄漏率测试。也就是拿训练中没见过的 `100 mL/min` 泄漏率来做闭环验证，测试集 7 组工况，当前 `RMSE = 8.600e-4`，`MAE = 3.548e-4`。这说明模型对“新流量”也能工作，但比新位置更难。

第四，已经对源参数做了初步定量评估。当前 `sensor` 参数回归 baseline 的结果是：源位置平均误差约 `214.29 mm`，最大误差约 `282.62 mm`；泄漏率 `MAE = 34.94 mL/min`，平均相对误差约 `10.19%`。这组数字说明泄漏率估计已经有一定可用性，但源位置定位还只能算“初步结果”，不适合在汇报中说得过满。

第五，已经做了“训练数据量增加后模型能力如何变化”的重复实验。当前用 `6/12/24/31` 四个训练规模，每个规模做了 3 次重复。结果不是简单的“数据越多越好”，而是出现了非单调波动：`12` 组训练样本时平均结果最好，`24` 组波动最大。这说明当前瓶颈不只是数据量，还包括训练稳定性、低泄漏率样本难度和观测注入机制本身。

## 可直接汇报的文字版

如果把当前阶段的结果压缩成一段老师容易快速判断的汇报文字，可以直接这样表述：本阶段已经不再只是停留在方法思路和示意图层面，而是已经在可控的 CFD 真值下完成了可量化的闭环验证。当前正式结论基于清洗后的 `48` 组有效 CFD 工况，而不是原始候选 `56` 组，因为其中 `8` 组原始浓度列异常为常数 `1`，不能代表真实扩散过程。基于这 `48` 组 clean 数据，已经完成了“CFD 真值 -> 传感器抽样 -> 模型反演 -> 回到 CFD 全场逐点对比”的完整验证链，说明现在的结果是可以被真值检验和定量评价的，而不是单纯生成了一些看起来像的图。

从主线结果看，模型已经具备初步泛化能力。对训练中未见过的泄漏位置 `(400,0,0)`，测试集 `7` 组工况的闭环结果为 `RMSE = 6.521e-4`、`MAE = 3.717e-4`；对训练中未见过的泄漏率 `100 mL/min`，测试集 `7` 组工况的闭环结果为 `RMSE = 8.600e-4`、`MAE = 3.548e-4`。这说明模型不仅能在训练条件下工作，也能够迁移到训练中未见的位置和未见的流量条件上，但未见泄漏率任务明显比未见位置更难，因此后续数据补充和模型优化应优先关注低流量和未见工况。

从模型能力和模型成本看，当前结果也已经能支撑进一步判断。传感器数量与观测时长敏感性实验一共完成了 `9` 组条件，当前最优条件是 `30` 个传感器、`20` 个观测时间步，对应 `RMSE = 5.324e-3`；如果按观测时长做平均，`20` 步、`60` 步和 `120` 步的平均 `RMSE` 分别约为 `5.921e-3`、`7.496e-3` 和 `8.744e-3`。这说明误差并没有随着传感器数量和观测时长增加而单调下降，当前瓶颈不只是“观测信息不够”，更在于观测注入机制本身还不够强。训练数据量实验方面，`6/12/24/31` 四个训练规模各做了 `3` 次重复，其中 `n=6` 的 `RMSE = 1.918e-3 ± 2.197e-4`，`n=12` 的 `RMSE = 7.361e-4 ± 4.647e-5`，`n=24` 的 `RMSE = 2.658e-3 ± 2.597e-3`，`n=31` 的 `RMSE = 1.180e-3 ± 5.446e-4`。这组结果表明，当前性能与训练数据量之间不是简单的线性提升关系，也不是稳定的边际递减，而是呈现明显的非单调波动；其中 `n=12` 是当前平均最优点，`n=24` 的高方差则暴露出训练稳定性和生成链路漂移问题。

从源参数估计看，当前已经形成了可以汇报的初步 baseline。基于 `sensor` 参数回归器，泄漏源位置定位的平均误差约为 `214.29 mm`，最大误差约为 `282.62 mm`；泄漏率估计的 `MAE = 34.94 mL/min`，平均相对误差约为 `10.19%`。这意味着泄漏率估计已经具备初步可用性，而泄漏源定位虽然已经能做定量评价，但精度仍明显弱于三维浓度场主线结果。因此，当前最稳妥的阶段性结论是：三维浓度场反演已经完成可验证、可量化的正式闭环验证，能够支撑“方法有效”的判断；泄漏率估计和源位置定位已经建立 baseline，但仍属于需要继续优化的次主线结果。

## 这件事的价值

这项工作的意义，不是再做一套“能生成漂亮图像的模型”，而是回答一个更接近工程真实需求的问题：在实际场景里，传感器数量往往很少，测点也很稀疏，我们能不能只凭少量局部浓度时间序列，反推出整个空间里的氢气浓度分布。

如果这个问题能被稳定解决，它至少有三层价值。

第一层是监测价值。现实中不可能在空间里布满传感器，所以真正有意义的不是“测到了几个点”，而是能不能从几个点恢复整个场。

第二层是诊断价值。如果不仅能重建浓度场，还能进一步估计泄漏位置和泄漏率，那么它就不只是“看到了异常”，而是接近“定位了问题并估计了严重程度”。

第三层是科研价值。导师这次强调的重点非常清楚，不是继续堆图，而是必须证明模型在可控真值下确实反演得对，并且用量化指标证明它到底对到什么程度。现在这一步已经做出来了，这意味着课题从“方法设想”进入了“有验证证据支撑的方法研究”阶段。

## 方案比较

接下来实际上有两条路线可以走。

路线一是先把当前小箱体场景做扎实，再扩展。具体做法是：继续在同一尺寸箱体内补更多低泄漏率、不同通风、不同障碍物的 CFD 工况；同时改进观测注入机制，并逐步把物理约束引入模型。  
这条路线的优点是结论更稳，能先把“这个方法在标准场景下确实有效”做扎实；缺点是短期内看起来没有那么“铺得开”。

路线二是直接快速扩展到更多尺寸、更多复杂场景，尽快做泛化展示。  
这条路线的优点是范围大、图更多；缺点是如果基础验证还不够稳，结果会很容易变成“场景很多，但每个结论都不够硬”。

当前更推荐路线一。原因很直接：老师现在最关心的是“你有没有在一个可控真值下证明它真的有效”，而不是“你能不能先堆出很多更复杂的场景”。只有先把闭环验证、定量指标和同尺度泛化讲扎实，后续再扩尺寸、扩障碍物、扩通风时，结论才站得住。

## 请求与下一步

现在最希望老师帮助判断的，不是“这个方向要不要做”，而是“下一阶段优先把哪件事做扎实”。

当前更建议老师支持下面这个推进顺序：

先继续在同尺寸小箱体上补足更系统的数据，尤其是低泄漏率、不同通风和障碍物工况，同时把观测注入机制和物理约束做强；在这个基础上，再进入尺寸稍大或稍小场景的真正泛化验证。

这样安排的原因是，当前已经证明了方法主线“能做、可验证、可量化”，但还没有资格说“已经稳定泛化到更复杂场景”。如果现在直接把重点切到大范围扩场景，短期虽然材料会变多，但每个结论都会变得不够硬。

因此，下一步建议明确分成三件事。

第一，继续补 clean 数据，重点补低泄漏率、通风变化和障碍物工况，优先保证数据质量和元数据完整，不再让异常 case 混入主线。

第二，继续强化三维浓度场主线，把“观测是否真正起作用”这个问题彻底做扎实。也就是后续不仅要比较传感器数量和观测时长，还要进一步增强观测约束本身。

第三，在主线更稳之后，再正式开始尺寸泛化实验。到那时，泛化结论会更可信，也更适合写进论文和正式答辩材料。
