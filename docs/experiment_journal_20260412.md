## 低泄漏率加权结论

`lowflow_focus_v1` 明显优于 `none` 和 `lowflow_balanced_v1`，尤其在 `n=6` 下整体指标和低泄漏率子集指标均最好。短期报告中可以将 `lowflow_focus_v1` 作为当前低泄漏率加权训练推荐方案。

需要注意的是，这一结论仍基于当前 56-case 数据池和 holdout 位置测试集。由于 50/100 ml/min 样本仍偏少，后续仍需要继续补低泄漏率 CFD，尤其是补齐缺失位置，否则低泄漏率精度结论容易受少数 case 影响。

## case_0049 异常排查与数据质量修正

按优先级排查 `case_0049` 后，已经确认该异常不是模型本身导致，而是数据张量异常。`holdout_rate_0100_val_0200/test.h5` 中 `case_0049` 的真值场几乎是全 1：

- `min = 0.99999988`
- `max = 1.00000012`
- `mean = 1.0`
- `sum = 13271040.0`

正常 100 ml/min case 的 mean 通常约为 `1.56e-4` 到 `1.59e-4`，因此 `case_0049` 不是合理浓度场。进一步检查总数据池 `cfd56_all_T120_interp48.h5` 后发现，异常不止一个 case，而是以下 8 个 case 都是全 1 张量：

| case_id | 位置/mm | 泄漏率/ml·min⁻¹ | 状态 |
| --- | --- | ---: | --- |
| case_0048 | (100, 0, 0) | 400 | 全 1，无效 |
| case_0049 | (200, 0, 0) | 100 | 全 1，无效 |
| case_0050 | (200, 0, 0) | 200 | 全 1，无效 |
| case_0051 | (200, 0, 0) | 50 | 全 1，无效 |
| case_0052 | (200, 0, 0) | 400 | 全 1，无效 |
| case_0053 | (200, 0, 0) | 600 | 全 1，无效 |
| case_0054 | (200, 0, 0) | 1000 | 全 1，无效 |
| case_0055 | (200, 0, 0) | 800 | 全 1，无效 |

这些异常 case 对当前实验的影响如下：

- `holdout_400_0_0_val_0200/test` 不含无效 case，因此未见位置测试集本身是干净的。
- `holdout_400_0_0_val_0200/train` 含 `case_0048` 一个无效训练样本。
- `holdout_400_0_0_val_0200/val` 的 7 个 `(200,0,0)` case 全部无效，因此该 val 位置不能用于任何有效结论。
- `holdout_rate_0100_val_0200/train` 含 6 个无效样本，`val` 含 1 个无效样本，`test` 含 `case_0049` 一个无效样本，因此未见泄漏率实验整体需要重建数据后重做。
- 训练数据量实验中，`n=6` 与 `n=12` 子集未包含无效 case；`n=24 repeat_01/repeat_02` 和全部 `n=42` 子集包含 `case_0048`，这很可能解释了大样本结果显著变差。

因此，之前“训练数据量增加未带来单调收益”的结论必须降级为“在当前含异常 case 的数据版本中观察到非单调波动”。不能把它作为干净的科研结论直接写进报告。更严谨的下一步是剔除或重新转换 `case_0048` 到 `case_0055`，构建 clean 数据集后，至少重跑训练数据量实验中受影响的 `n=24/n=42` 子集，以及未见泄漏率实验。

## 源参数结果冻结策略

源参数部分短期只采用 `sensor` 回归器作为正式基线，不再把 `core` 或 `hybrid` 作为主结果。原因是 `core/hybrid` 在测试集上出现严重泛化失败，而 `sensor` 模型虽然源位置误差仍偏大，但泄漏率估计相对稳定，且符合“由传感器输入直接估计泄漏源位置和泄漏率”的工程闭环。

当前报告中的建议表述为：泄漏源位置与泄漏率估计已从重建场峰值搜索升级为基于传感器时序特征的参数回归基线；当前 `sensor` 回归器在未见位置测试集上源位置平均误差约 `217 mm`，泄漏率 MAE 约 `34.94 ml/min`，泄漏率相对误差约 `10.19%`。`core/hybrid` 特征当前过拟合，暂不作为正式结论。
