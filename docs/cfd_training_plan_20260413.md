# 氢泄漏 CFD 数据积累与后续训练规划（2026-04-13）

## 1. 这份文档的用途

这份文档不是阶段性结果汇报，而是后续数周到数月内的数据生产、训练组织和论文推进执行稿。目标是解决三个问题：第一，接下来 CFD 应该优先补什么工况，而不是继续随机补；第二，新增数据进仓库前必须满足什么质量标准，避免再次出现异常列污染主线结果；第三，模型后续要怎样从“能在同一箱体上反演”推进到“有更强泛化性和更明确物理含义”。

## 2. 当前已经可以交付给导师的内容

当前已经具备阶段性可交付结果。基于 `cfd48_clean`，已经完成 clean 数据闭环验证：`CFD 真值 -> 传感器抽样 -> 模型反演 -> 回到 CFD 全场逐点对比`。同时已经有正式量化指标，不再只是图像展示。未见位置 `(400,0,0)` 的 test 指标为 `RMSE = 6.521e-4`、`MAE = 3.717e-4`、有效羽流区相对 `L1 = 1.379`、质量积分相对误差 `0.899`；未见泄漏率 `100 mL/min` 的 test 指标为 `RMSE = 8.600e-4`、`MAE = 3.548e-4`、有效羽流区相对 `L1 = 2.050`、质量积分相对误差 `0.932`。此外，训练数据量多次重复实验也已经完成，可以支撑“数据量与性能关系并非简单单调”的阶段性讨论。

当前不能直接对导师宣称的内容也要提前写清楚。现在还不能说模型已经完成严格意义的尺寸泛化验证，因为还没有新的箱体尺寸数据。也不能说泄漏源位置和泄漏率联合反演已经进入稳定可用状态，因为当前源参数部分虽然 `sensor` 回归器可作为基线，但主线成果仍然是浓度场反演。还不能说模型已经在真实实验域上完成强外推验证，因为真实实验数据尚未作为正式外部盲测集纳入主线论证。

## 3. 当前数据状态与核心问题

当前正式主线应统一以 `cfd48_clean` 为准，而不是继续使用旧的 `cfd56`。原因是原始 `56` 组候选数据中有 `8` 组 case 的原始浓度列异常，表现为 `molef-h2` 近似恒定 `1`，已经证明会污染训练数据量实验和未见泄漏率实验。因此，后续所有主线训练、评估、图表和导师汇报都应以 clean 数据为唯一标准。

当前最主要的工程问题也已经比较明确，不再是“数据能否转成张量”或“流程是否能跑通”，而是两点。第一，数据矩阵仍然不够系统，尤其是同一箱体尺度下的位置、泄漏率和低泄漏率样本覆盖还不够整齐。第二，生成链路的稳定性和观测注入强度仍然不足，这也是 `n=24, repeat=1` 出现系统尺度漂移的直接背景。后续工作必须同时从“数据组织”和“模型物理约束”两端推进。

## 4. CFD 数据积累的总体原则

后续数据不能继续按“哪里还缺一点就补一点”的方式推进，而要按变量分层控制。原则上一次只新增一类变化因素，优先保证可比较性。最合理的顺序是：先补齐同尺度、无障碍、单一通风边界下的位置和泄漏率覆盖；再引入通风强度变化；再引入障碍物；最后进入箱体尺寸变化。这样每一层都能单独回答一个科研问题，而不是把多个变化因素混在一起导致结论无法归因。

另一个原则是，后续所有新增 case 都必须服务于评估问题。也就是说，新增数据不仅要能训练，还要能支撑 holdout 设计。如果准备验证“未见位置泛化”，就必须有足够多的位置，且测试位置不能只是一两个孤立点；如果准备验证“未见泄漏率泛化”，就必须有完整的泄漏率档位覆盖；如果准备验证“尺寸泛化”，就必须至少有两个新的尺寸等级，而且这些新尺寸下仍要保有基础位置和泄漏率矩阵。

## 5. 第一优先级：补齐同尺度基准数据池

当前箱体尺寸固定为 `1.0 m × 0.8 m × 0.8 m`，泄漏口位于底部平面。建议把底面位置系统化为规则网格，而不是继续靠零散位置堆数据。优先建议的底面位置集合为 `x ∈ {0, 100, 200, 300, 400} mm` 与 `y ∈ {0, -150, -300} mm` 的组合，共 `15` 个位置，`z` 固定为当前泄漏高度。泄漏率继续采用 `50, 100, 200, 400, 600, 800, 1000 mL/min` 共 `7` 档。这样构成一个完整的同尺度标准基准矩阵，理论总规模是 `15 × 7 = 105` 组。

这 `105` 组并不是随便给出的数字，而是当前最合适的第一阶段目标。它足够大，能支撑未见位置和未见泄漏率实验，同时仍在可管理范围内。与当前 `48` 组 clean 数据相比，意味着还需要新增 `57` 组有效 clean case。若时间和算力不足以一次补满，最小可执行目标是先达到 `84` 组，即至少保证 `12` 个位置全覆盖 `7` 个泄漏率。这个阶段不建议加入障碍物和尺寸变化，否则会把最基础的位置和泄漏率规律打散。

在同尺度补数时，低泄漏率必须额外重视。`50` 和 `100 mL/min` 样本的羽流更弱、信号更稀疏、对传感器噪声更敏感，也是当前最容易被高泄漏率平均指标掩盖的部分。因此，除了完整保留 `50/100/200/400/600/800/1000` 七档外，建议新增数据优先补低泄漏率缺口，必要时甚至允许低泄漏率比高泄漏率多一层重复，以支撑单独的低泄漏率分析。

## 6. 第二优先级：通风条件扩展

在同尺度位置和泄漏率矩阵基本成型之前，不建议马上大规模加入障碍物。但可以先把通风条件作为第二个变量引入。推荐做法是，在已经较稳定的同尺度基准场景中选 `6` 个代表位置，覆盖中心、近壁、近出口和对角区域；再选 `4` 个代表泄漏率 `50/100/400/1000 mL/min`；在 `3` 档通风条件下组合，形成 `6 × 4 × 3 = 72` 组通风扩展数据。

通风条件的设置不要太碎，最好是明确的三档，例如低通风、基准通风、高通风。每一档都必须在 manifest 中保留可直接使用的数值标签，如入口风速、换气次数或边界流量。这样后续既可以做条件泛化，也可以将其作为显式条件输入模型，而不是只在文件名里隐式保存。

这一层数据的科研作用主要是回答两个问题。第一，背景流场变化是否会明显削弱基于稀疏测点的反演稳定性。第二，后续物理约束中对流项的作用是否真正能在变通风条件下体现出来。如果这一层数据做得足够规整，后续论文里可以单独形成一组很有价值的实验。

## 7. 第三优先级：障碍物布局扩展

障碍物扩展必须晚于通风扩展，因为障碍物本质上会改变流场拓扑。如果在基准位置和基准通风规律还没有摸清楚之前就加入障碍物，后续误差来源很难区分。建议障碍物阶段先做少量但典型的布局，而不是一开始设计很多复杂几何。推荐至少设置三类：无障碍、单立方体障碍、近出口障碍。必要时再增加一类近源障碍。

在数据规模上，建议先做 `3` 类障碍布局、`4` 个代表位置、`4` 个代表泄漏率，共 `48` 组。这里仍然推荐保留 `50/100/400/1000 mL/min`，因为这四档已经能覆盖弱、中、强泄漏区间。障碍物的大小、位置、距壁距离必须进入 manifest，不然后续没法做条件学习或做误差分层分析。

## 8. 第四优先级：尺寸泛化数据

严格意义上的尺寸泛化，必须依赖新的几何尺度数据，而不是依靠同一箱体内的插值或位置外推来解释成尺寸泛化。建议至少准备两个新尺寸，一个缩小尺度，一个放大尺度。例如可考虑 `0.8 × 0.64 × 0.64 m` 和 `1.2 × 0.96 × 0.96 m` 两档，与当前 `1.0 × 0.8 × 0.8 m` 构成三尺度体系。比例是否完全等比可以根据实际设备尺寸再定，但一定要让尺度变化明确且可复述。

在第一轮尺寸泛化数据中，不必追求很大规模，但必须保持基础矩阵完整。推荐每个新尺寸至少做 `4` 个位置、`4` 个泄漏率、`2` 档通风，共 `32` 组；两个新尺寸就是 `64` 组。这样虽然还不是很大，但已经足够形成“在未见尺寸上测试”的正式实验。如果时间更紧，可以把最小集压缩到每个新尺寸 `16` 组，但那样只能做早期摸底，不适合正式汇报。

## 9. 建议的数据总量里程碑

为了便于后续排期，可以把数据积累目标拆成三个里程碑。里程碑一是同尺度 clean 基准池达到 `84` 组，这是最小可训练、可分层、可做 holdout 的规模。里程碑二是同尺度标准基准池达到 `105` 组，并完成通风扩展到至少 `177` 组，即 `105 + 72`。里程碑三是在此基础上加入障碍物和尺寸数据，总规模推进到 `250` 组以上。到这个阶段，才真正具备讨论“泛化性”和“物理约束是否提高跨场景鲁棒性”的底气。

如果近期时间很紧，建议把阶段性目标压缩为：先把同尺度 clean 数据做到 `84` 到 `105` 组，再补一个最小尺寸泛化集 `32` 组。也就是说，优先把总 clean 数据规模推到 `116` 到 `137` 组之间。这个规模已经足够支撑下一轮比较像样的论文结果。

## 10. 每个 CFD case 必须记录的元数据

后续每个 case 在进入张量转换之前，manifest 至少要有以下字段：`case_id`、原始目录名、箱体尺寸 `size_x/size_y/size_z`、泄漏源坐标 `source_x/source_y/source_z`、孔径、泄漏率、通风入口速度或流量、出口条件、障碍物编号、障碍物几何参数、仿真总时长、时间步长、浓度列名、坐标列名、原始文件哈希、张量转换参数、质检标记和备注。

建议再额外保留两个非常实用的字段。第一个是场景标签，例如 `base_box`、`vent_high`、`obstacle_near_outlet`、`size_large`，方便后续直接做筛选。第二个是清洗来源，记录该样本是否经过原始列修复、是否有重采样、是否存在缺帧补齐。只要这些信息缺失，后续就很容易再次出现“训练结果异常但难以回溯”的情况。

## 11. 张量转换与原始数据删除规则

原始 ASCII 数据不能在“张量已经生成”之后马上删除。至少要满足四个条件，才可以清理原始文件。第一，`manifest.csv`、`report.json`、`meta.npy` 和 HDF5 已完整生成。第二，HDF5 能被脚本打开且形状正确。第三，经过 `validate_hdf5_quality.py` 检查后 `flagged_count=0`，且不存在浓度列恒定异常。第四，随机抽检若干 case，确认 HDF5 中重建出的时间序列与原始 ASCII 一致。

更稳妥的做法是保留一个原始文件哈希 + manifest + report 的索引层。即便原始 ASCII 被删除，也能在将来知道当时用的是哪批原始数据、经过了什么转换参数、是否有质检异常。如果没有这层索引，后续再发现问题时就会失去追溯能力。

## 12. 后续训练应怎样组织

短期内不建议继续盲目扩充超参数扫描，而应该先把 clean 数据、固定测试集和评估流程冻结。训练主线建议分成两个层次。第一层是稳定基线层，包括 FTM、GPSD、MPDPS 和 `sensor` 源参数回归器。这一层的主要任务是给出可复现、可比较的基线。第二层是改进层，只改一个关键因素，例如观测注入机制、非负约束、质量约束或 PDE 残差，而不是一次同时改很多部分。

后续每次新训练都应该围绕固定问题展开。比如，如果当前目标是回答“更多数据是否有帮助”，就冻结模型结构，只扩数据量并做重复实验；如果当前目标是回答“物理约束是否有帮助”，就冻结数据集和 split，只比较损失函数变化。只有这样，结果才真正能解释。

## 13. 建议加入的物理规律

当前最值得优先加入的物理规律不是复杂的大而全模型，而是三类最容易落地、最能解决现有问题的约束。

第一类是非负性与尺度约束。浓度场不能为负，因此可以在输出端加入 `softplus` 或用损失项 `L_nonneg = mean(ReLU(-ĉ)^2)` 直接惩罚负值。同时可以加入全场质量尺度约束，令积分质量与真值接近，避免出现 `n=24, repeat=1` 那种整体符号与尺度漂移。

第二类是对流扩散约束。若 CFD 中能导出背景速度场 `u`，则可以用对流扩散方程残差作为弱监督：

```text
L_pde = mean | ∂ĉ/∂t + u·∇ĉ - ∇·(D∇ĉ) - sθ |²
```

这里 `ĉ` 是预测浓度场，`D` 是有效扩散系数，`sθ` 是泄漏源项。即使 `D` 不是严格已知，也可以先把它视作条件参数或小范围可学习参数。

第三类是边界与源项一致性约束。泄漏源附近的源项应该局部集中，壁面附近不应出现明显不合理的负通量，出口附近的通量变化应与通风边界条件相匹配。即便暂时不做严格边界方程，也可以把这些条件转成软约束项，先提升生成链路稳定性。

## 14. 推荐的总损失形式

后续可以考虑把训练目标逐步写成下面这种形式，而不是继续完全依赖纯数据重建：

```text
L_total =
  λ_recon * L_recon
  + λ_sensor * L_sensor
  + λ_nonneg * L_nonneg
  + λ_mass * L_mass
  + λ_pde * L_pde
  + λ_bc * L_bc
  + λ_src * L_src
```

其中 `L_recon` 是全场重建误差，`L_sensor` 是传感器一致性误差，`L_nonneg` 是非负约束，`L_mass` 是全场质量或积分尺度约束，`L_pde` 是对流扩散残差，`L_bc` 是边界一致性约束，`L_src` 是源位置和泄漏率监督项。第一阶段不需要一次把所有项都加进去，建议先从 `L_nonneg + L_mass` 开始，再逐步加入 `L_pde`。

## 15. 传感器与观测实验怎样做才有价值

导师要求的“传感器数量”和“观测时长”影响实验，不能在观测注入机制还不稳定时反复重跑。等观测注入策略稳定后，建议固定测试集，只做单因素实验。传感器数量建议取 `4/6/8/12` 四档；观测时长建议取 `10/20/40/60/120 s` 五档。所有实验都必须保持相同测试集、相同随机种子集合和相同评价脚本，最后报告 `mean ± std`，不能只给单次运行结果。

如果后续引入真实实验数据，还要额外记录传感器响应延迟、零点偏差和量纲换算方式。否则“传感器数量影响”实验很容易被传感器质量差异掩盖。

## 16. 训练数据量实验的正确做法

当前已经证明，训练数据量实验必须做“多次重复的分层子集实验”，不能只随机抽一次。以后数据池扩大后，这条原则不能变。每个数据量档位至少做 `3` 次不同随机子集，并尽量保持位置、泄漏率和低泄漏率样本占比一致。建议未来把档位推进到例如 `12/24/48/72` 或 `24/48/72/96`，而不是继续停留在极小样本区间。

如果出现“数据更多反而更差”的情况，排查顺序应固定为：先看数据质量，再看子集分布，再看训练是否稳定，最后才怀疑模型容量。这样可以避免再次把数据污染问题或生成链路不稳定问题误判为“更多数据无用”。

## 17. 真实实验数据怎么融入

真实实验数据不应在数量很少时直接和 CFD 数据混在一起大规模训练。更合理的做法是先把 CFD 用作主训练域，真实实验数据用作校准域和外部验证域。若近期只能拿到 `8` 到 `12` 组真实实验，建议 `2` 到 `4` 组用于传感器偏差和量纲校准，其余全部作为盲测，不参与主模型训练。这样才能真正回答“模型是否有工程外部有效性”。

当真实实验数据规模提升到 `20` 组以上时，再考虑引入简单的域适配策略，例如输入标准化校准、传感器偏移校准、少量真实数据微调或对抗式域对齐。但在现阶段，保留真实实验作为高价值测试集通常比混入训练更重要。

## 18. 推荐的执行顺序

后续工作建议按下面顺序推进，而不是同时摊开。

1. 冻结当前 `cfd48_clean` 主线结果，作为导师汇报版本。
2. 补齐同尺度 clean 基准池，目标至少 `84` 组，理想 `105` 组。
3. 每批新增数据先做原始列核查和 HDF5 质检，再入库。
4. 在冻结 split 的前提下，先加最简单的非负和质量约束，验证是否能抑制系统尺度漂移。
5. 再做观测数量和观测时长影响实验。
6. 再做扩大量级的训练数据量重复实验。
7. 完成通风扩展与障碍物扩展。
8. 最后启动尺寸泛化数据和尺寸迁移实验。

## 19. 一句话版的下一阶段目标

如果要把后续工作压缩成一句话，那么下一阶段的正确目标不是“再多训一点”，而是“把同尺度 clean 数据池做完整、把观测注入和物理约束做扎实、再用新增通风和尺寸数据把模型从阶段性可验证原型推进到真正可推广的研究原型”。
